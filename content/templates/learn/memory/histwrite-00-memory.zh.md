# Histwrite-00 写作宪法（生成阶段 / 短提示词优先）

> 目标：让“起草阶段”的提示词保持**短、正向、稳定**；把细颗粒的禁令与形态学指标（1/2/3/4/6）迁移到**精修/回归**阶段执行。  
> 适用：中文历史学论文（严肃学术稿），材料密集但以论证为主线。  
> 默认取向：**不省 token，优先一次到位的可提交质量**（但每轮必须有明确主攻问题与验收标准，禁止全文乱改）。

---

## 1) 分阶段加载（非常重要）

把“记忆”当作一套分层系统，而不是一份越补越长的禁令清单。

- **Clarify / 蓝图（澄清意图、界定范围）**  
  - 必读：本文件（00）+ `histwrite-10-workflow.zh.md`  
  - 目标：把研究对象、范围边界、竞争解释与证据缺口写清楚。

- **Draft / 起草（把“论”写稳）**  
  - 写作约束：只加载本文件（00）+ `histwrite-01-style-profile.zh.md`（若存在）  
  - 禁止做法：把 1/2/3/4/6 的“雷区清单”整段塞进起草 prompt（会诱发碎句化、平铺化、用短句躲触发器）。  
  - 目标：论题集中、段落推进顺、史论结合不失衡、学术史对话到位。

- **Polish / 精修（把“形态学 AI 痕迹”打散）**  
  - 必读：00 + 01 + `histwrite-40-deai-polish-playbook.zh.md`  
  - 评测口径：以 `.histwrite/learn/rubrics/deai-flow.zh.md` 的 1/2/3/4/6 为主。

- **Eval / 回归与调优**  
  - 只用 rubrics（`deai-flow.zh.md`、`draft-quality.zh.md`）打分/断言，别再把细则混回起草提示词里。

> 原因：起草阶段最缺的是“论证链条”，不是“连词降频”。先把论证写稳，再用精修把形态学痕迹收拾干净，质量更可控。

## 2) 起草阶段的硬原则（宪法条款）

这些条款是“少而不冲突”的总原则：它们应该**长期稳定**，不因某一版的小问题而不断加长。

### 2.1 学术论文的严肃性（底盘）

- 写作对象是学术论文，不是科普、随笔或讲故事。  
- 每个判断要可追责：概念界定清楚、时间线稳定、推论克制。  
- 降低黑话/名词化/翻译腔是为了**更清楚**，不是为了“口语化”。

### 2.2 史论结合（史料只能服务于“论”）

- 史料是论证支点，不是堆砌对象。  
- 一段要组织成解释链：**判断句 → 证据链（2–4 份材料共同支撑）→ 机制解释 → 转场回扣主题**。  
- 避免机械循环：“每份史料一小段解读 → 下一份史料”。材料再多也要被组织进同一条解释线。

### 2.3 线性叙述（呼吸感）

- 读感必须是一条连续叙述线，不是清单拼贴、不是说明书块。  
- 不依赖二级标题切分论证；用段落内部推进完成“一口气论述完一个观点”。  
- 允许段落内节奏变化：短句推进、长句承载解释，但不要“句型固定 + 均匀节拍”。

### 2.4 学术史（只用二手，写“谱系—分歧—缺口—引出切口”）

- 学术史只引用二手研究（论文/专著/博论等），不得用一手史料充当学术史证据。  
- 学术史写法要“对话化”：谁怎么解释、分歧在哪、盲点是什么，你的切口解决哪类解释难题。

### 2.5 论题边界（避免以偏概全，尤其是“接受史”写法）

- 不要用少数右翼刊物/精英通信去代表“美国公众总体”；必须反复声明研究对象的范围（例如：保守主义运动内部、右翼话语中的接受与再解释）。  
- 民调（NORC/Roper 等）属于边界条件：可以用来说明激进主张在国内政治上推进到哪里会碰壁，而不是把它当作论证软肋。

### 2.6 中文学术句法（允许紧凑复句，反对碎句化）

- **允许并鼓励紧凑复句**：只要层级清楚、限定关系明确，就不要为了“去 AI”把一句拆成三句平铺短句。  
- 拆句只用于：英式链式修饰导致读者回溯，或同构模板近距离堆叠且无法在句内改造时。  
- 术语尽量中文化；正文不夹英文词（必要时首次括注英文，后续只用中文）。  
- 避免生造概念与空泛黑话（如“重构/嵌入/张力”“xxx性”堆叠）；要能落到“谁做了什么、导致什么、被谁如何理解”。  
- 引号只用于直接引文、确需保留的专名/口号或约定俗成概念；不要用引号包裹自造概念。
- 列举不是原罪；问题是“裸列名词串”成为句子骨架或在相邻段落反复出现形成节拍（读者看完仍不知道关系/作用）。仅在这种情况下才改写：优先用动作/过程/场景承载关系；不要见到顿号就强行给每一项补“做了什么”，以免扩写与清单腔回潮。

### 2.7 学术规范（门面硬约束）

- 引注体例二选一并全篇一致（《历史研究》或 Chicago Notes-Bibliography）。  
- 脚注/参考文献中**本地路径必须为 0**；档案引用写清 Repository/Collection/Box/Folder/Item 等字段。  
- 学术史段落引用只来自二手研究；转引需标“转引自”（转引不计入一手数量）。
- 定稿前必须单独跑一次 `finalcheck`；本项目默认把“占位符清零、脚注与参考文献自洽、体例错误清零”视为门面硬约束。

## 3) 写作硬指标（默认基线）

> 若导师/课程另有要求，以最新口头要求为准；但默认遵守以下基线。

- 总字数：**12000–15000**（中文）  
- 一手史料：**≥35 条**（必须服务论证，不以堆砌为荣）  
- 二手文献：**≥20 条**（学术史与关键论证需足量对话）
