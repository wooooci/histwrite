# Histwrite-00 写作宪法（精简版 / 仅供低限额接口）

> 用途：当第三方 OpenAI-compatible 网关存在“单次请求体积/时延”限制时，用本精简版作为**起草阶段**约束。  
> 注意：本文件刻意不包含 1/2/3/4/6 的细颗粒“雷区清单”，避免诱发碎句化；精修/回归时请改读 `histwrite-40-deai-polish-playbook.zh.md` 与 rubrics。

## 1) 分阶段加载（必须遵守）

- Draft 起草：只加载本文件 +（可选）`histwrite-01-style-profile.zh.md`  
- Polish 精修：再加载 `histwrite-40-deai-polish-playbook.zh.md`；评测以 `.histwrite/learn/rubrics/deai-flow.zh.md` 为准  
- Eval 回归：用 rubrics 打分/断言，不把细则混回起草 prompt

## 2) 起草硬原则（少而不冲突）

- 学术严肃性：概念清楚、时间线稳定、推论克制；不写科普/随笔。  
- 史论结合：史料只做论证支点；每段 = 判断句 → 证据链（2–4 份材料共同支撑）→ 机制解释 → 转场回扣主题。  
- 线性叙述（呼吸感）：一口气论述完一个观点；不靠清单拼贴或标题堆叠。  
- 学术史只用二手研究：写“谱系—分歧—缺口—引出切口”。  
- 中文句法：允许紧凑复句；禁止为合规把一句拆成三句平铺短句；避免黑话/“xxx性”堆叠与正文英文夹杂。  
- 学术规范（门面）：体例统一；本地路径=0；档案字段齐全；转引标“转引自”；定稿前必须额外跑一轮 `finalcheck`。

## 3) 默认硬指标

- 12000–15000 字；一手 ≥35；二手 ≥20。  
- 不省 token（默认）：允许更多上下文、候选与自检，但每轮必须有明确主攻问题，禁止全文乱改。
