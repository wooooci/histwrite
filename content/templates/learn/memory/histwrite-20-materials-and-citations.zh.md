# Histwrite-20 材料入库 / OCR / 元数据与引注（门面工程）

> 用途：让“材料真的能用、引用真的干净”成为硬约束。

## 1) 文件与目录：先可追溯，再谈效率

- 原始下载：`材料/_downloads/`（不直接写入正文）
- OCR 输出：`材料/_ocr/`（docx/txt；扫描版 PDF 先 OCR）
- 可检索索引：`材料/_index/`（提取后的纯文本 + library.json + 快照等）
- 建议增加二级分类（按材料性质）：
  - `材料/一手/`：档案、报刊、广播稿、内部备忘录、书信、民调等
  - `材料/二手/`：论文、专著、博论、研究综述

命名建议（不要求统一，但要稳定）：
- `YYYYMMDD_机构或作者_标题关键词_来源库_页码范围.pdf/docx`
- 同一材料多版本（原始/截图/OCR）保留同一“主键”（同名前缀），便于映射。

## 2) “一手/二手”硬边界

- 学术史只允许引用二手研究。
- 一手史料只用于：支撑判断、呈现语境、提供细节、检验二手解释。
- 二手文献不只做背书：可标注“某观点/机制来自哪位作者”，形成对话关系。

## 3) 元数据模板（先占位，后补齐）

### 3.1 档案（推荐字段）

- Repository（馆藏机构）
- Collection（馆藏/基金/系列）
- Box / Folder（盒号/文件夹）
- Item（文件编号/标题）
- Date（日期）
- Creator/Author（形成者）
- Format（信件/备忘录/报告/传单/剪报等）
- Pages（页码范围，若只下载部分必须写清）

### 3.2 报刊/期刊（推荐字段）

- 刊名/报纸名
- 日期
- 版面/栏目标识
- 标题/作者
- 页码

### 3.3 民调/统计（推荐字段）

- 机构（如 NORC/Roper 等）
- 调查时间
- 样本/方法（若可得）
- 问题原文（必要时原文+中文译）
- 关键数据（百分比/趋势）
- 数据出处（报告/数据集编号/链接）

## 4) 引注清洗：本地路径必须为 0

- 禁止出现任何本地路径、索引路径、导出路径（例如用户目录、tropy_mineru 等）。
- 对档案类引用，必须改写为标准档案格式（Repository, Collection, Box, Folder...）。
- 对数据库下载的论文/博论：引用应指向标准书目信息（作者、题名、期刊/学校、年份、页码）。

## 5) 体例统一（从一开始就锁死）

- 二选一并全篇一致：
  - 《历史研究》体例（中文史学常用）
  - Chicago Notes-Bibliography（英文学术常用）
- 同类材料字段顺序、标点、缩写必须一致。
- **脚注区**与**参考文献区**要分开理解：
  - 脚注区强调“首次引注信息充分、页码/出处具体、同类格式一致”。
  - 参考文献区允许采用更典型的 bibliography-style 写法，尤其是英文专著、英文期刊文、学位论文、报纸、档案、传单/小册子、报告、连续出版物、照片等。
- 不要把“参考文献区不是脚注句法”误判成格式错误；真正要抓的是字段缺失、作者/机构/缩写前后错配、占位符、本地路径、体例混用。

## 6) 转引规则

- 二手文献中的一手史料可以转引，但必须标注“转引自”。
- 转引不计入“一手史料数量”。

## 6.5) 英文材料与档案条目的实践口径

- 英文档案/文件类材料并不都长得像“标准书”；题名 + 文献类型 + 日期（或 undated）+ 馆藏/出处信息，往往已经构成可接受条目。
- `Repository / Collection / Box / Folder / Item No. / Document No. / Gale / RFE / reprinted from` 这类字段在档案与数据库材料中属于来源链的一部分，不应被误当成噪音。
- 英文报纸、学位论文、连续出版物、照片、宣传册/小册子要按各自文类处理；不要为了“看起来整齐”强行套进专著或期刊模板。

## 7) 质量自检（提交前必跑）

- 脚注/参考文献：本地路径=0
- 学术史段：是否只引二手
- 关键档案：Repository/Collection/Box/Folder 字段是否缺失
- 同类条目：字段顺序与标点是否一致
- 跑一次 `finalcheck`：重点看“占位符、本地路径、脚注定义/使用是否一一对应、脚注区错误、参考文献区错误”
- 本项目交稿前默认目标：`historyStudyStyleFailures = 0`，并尽量把报告压到 `错误 / 警告：0 / 0`
