# Histwrite-10 工作流（对齐你的历史论文写作方法）

> 用途：把“灵感 → 学术史 → 材料检索 → 史料解读筛选 → 蓝图 → 大纲 → 分段写作 → 多轮精修 → 评测/标注 → 调优”变成可重复执行的步骤。
> 适用：中文历史学论文（严肃学术稿件），以“史论结合”为核心。

## 0) 开始条件（每次开工必做）

- 先读完 `.histwrite/learn/memory/` 下所有 `histwrite-*.md`（本目录即长期记忆约束）。
- **提示词分层（防碎句化）**：起草阶段只用 `histwrite-00-memory.zh.md`（宪法）+ `histwrite-01-style-profile.zh.md`（画像）做约束；1/2/3/4/6 的细颗粒规则只在精修/回归阶段启用（见 `histwrite-40-deai-polish-playbook.zh.md` + rubrics），不要塞进起草 prompt 逼模型走“拆句逃避”的合规路径。
- 本项目默认**不省 token**：优先一次到位的完成度与可提交性（允许更长上下文、更多候选、更多自检），但不允许因此牺牲论证的克制与可追责性。
- 明确：题目/研究对象范围、预期字数、引注体例（《历史研究》或 Chicago Notes-Bibliography）。
- 明确角色分工：你负责灵感、方向裁决、事实核对与最终审读；机器人负责流程执行、材料组织、草稿生成与回归检查。

## 1) 灵感 → 可做性（选题能不能做）

1. 记录灵感触发点：来自哪本书/哪篇论文/哪段材料；它引出的疑问是什么。
2. 同步做“可做性三问”：
   - 学界有没有争议点/分歧谱系可写？（不写“常识”）
   - 你能找到足够的一手材料吗？（类型与可得性）
   - 这个切口能否写出机制解释，而不是素材堆砌？
3. 在此阶段与模型对话的目标：逐步澄清意图（研究问题、时间跨度、核心概念界定、潜在反方）。

交付物：`蓝图/题目-问题清单.md`（可做性结论 + 下一步检索计划）。

## 2) 二手研究优先：学术史（只用二手）

1. 同时做中文与外文二手检索（示例渠道：CNKI、JSTOR、ProQuest 博论等）。
2. 学术史写作只允许引用二手研究：
   - 先梳理研究传统与关键分歧（谁和谁争什么）。
   - 再指出错位/盲点/缺口（为什么还值得写）。
   - 最后落回你的切口（你要解决哪一类解释难题）。

交付物：`蓝图/学术史-谱系与缺口.md` +（可选）`材料/二手研究清单.md`。

## 3) 蓝图后：材料查找计划（明确“去哪里找什么”）

1. 先从二手研究反推材料路径：
   - 他们用的档案馆/馆藏/盒号/文件夹类型是什么。
   - 他们在数据库里用的关键词、主题词、机构名、人名、项目名是什么。
2. 再做你自己的材料清单：
   - 必须材料（没有就写不下去的关键证据）。
   - 辅助材料（用于补充背景、细节、反驳）。
   - 约束材料（如民调/统计，提供边界条件）。
3. 下载策略（避免抓全本）：
   - 对超长档案：只截取相关页；同时抄录元数据（来源字段）以便引注。

交付物：`蓝图/sources-plan.md`（按“库/档案馆 → 查询式 → 预期命中材料类型 → 判断标准”列出）。

## 4) 入库 → OCR → 结构化元数据

1. 资料入库优先走文献管理与元数据抓取（例如 Zotero Connector / 本地 connector）。
2. 下载后统一走 OCR（例如 MinerU API）把扫描件变成可检索文本（docx/txt）。
3. 每份材料必须落一份“引用信息模板”（先占位，后补齐字段）：
   - 档案：Repository / Collection / Box / Folder / Item / Date / Author / Title
   - 报刊：报纸名 / 日期 / 版面 / 栏目 / 标题 / 作者
   - 民调：机构 / 时间 / 样本 / 问题原文 / 关键数据

交付物：`材料/_index/library.json` + `蓝图/library_index.md`（索引摘要）。

## 5) 史料解读与筛选（可批量）

目标不是“逐条复述史料”，而是把史料变成可用于论证的证据卡（evidence card）。

每份候选材料至少产出：
- 摘要（不超过 200–400 字）
- 可用性判断：是否直接支撑论文主题（是/否/待定）
- 可用于论证的要点（3–6 条）
- 语境/偏误提醒（立场、受众、机构目的）
- 元数据字段（供引注）
- 建议放置位置：最适合支撑哪个论点/哪一段

筛选规则：一批材料解读完成后，按主题相关度与论证价值分为：
- 必用
- 可用（备用）
- 不用（可删除或冷藏）

交付物：`材料/_index/evidence_cards/*.md`（或同等结构化卡片）。

## 6) 蓝图（机制解释优先）

蓝图必须包含：
- 核心问题、概念界定、时间分期
- 竞争解释（至少 1–2 条）与反方处理
- 证据地图：每条关键论点对应哪些类型的一手/二手证据
- 缺口清单：【缺口】明确标注需要补证的点（不要编造）

交付物：`蓝图/blueprint.md`。

## 7) 大纲（论证链，而不是标题树）

大纲按“段落推进”组织：每段写清楚
- 判断句（你要证明什么）
- 证据链（2–4 份材料组织到一个判断）
- 解释机制（为何这些证据能推出该判断）
- 转场方式（怎么接到下一段）

交付物：`大纲/outline.md` + `大纲/materials-to-outline.md`（材料分配表）。

## 8) 写作（按段落推进，保持连贯）

- 先写学术史段（只用二手），再写正文段。
- **边写边修（局部轻修）**：每写完一个部分，立刻做一次“门面 + 去 AI 硬门槛（1/2/3/4/6）”的快速回归，只改本部分，不动全文结构（打法见 `histwrite-40-deai-polish-playbook.zh.md`）。注意：这是写完后的小型精修，不要把细则塞进“起草提示词”。
- 每个部分写完，立刻产出“部分摘要 + 未解决问题清单”，供下一部分衔接。
- 避免“每份史料一小段解读”的机械循环：一段应组织多份材料共同支撑同一判断。

交付物：`正文/*.md`（分部/分章）+ `导出/draft.md`（合并稿）。

## 9) 多轮精修（最多 3–5 轮，每轮只打 1–2 个指标）

- **不省 token ≠ 全文乱改**：每轮仍只聚焦 1–2 个指标（或 1 类结构问题），否则极易风格漂移、论证链被打断。
- 结构轮：论点是否集中，是否跑题，段落推进是否顺。
- 证据轮：史料是否服务论证；是否出现堆砌；是否存在“该写论述却只有材料复述”。
- 文风轮（去 AI）：优先回归 1/2/3/4/6（连接词骨架/句式模板/冒号块/并列名词串/被字句）；其余（引号、黑话、其指代等）只在不牺牲论证严肃性时处理。具体精修打法见 `histwrite-40-deai-polish-playbook.zh.md`。
- 学术规范轮：脚注清洗（本地路径=0）、体例统一、档案字段补齐。

交付物：`导出/final.md` + `导出/revision_notes.md`。

## 10) 评测/标注/调优（写作长期进化）

- 评测：用固定 rubric 做回归（去 AI、草稿质量、史论结合等维度）。
- 标注：把你的逐句修改/批注沉淀成可学习数据（偏好规则 + 正反例）。
- 调优：先 prompt 调优（DSPy/Promptflow/promptfoo 回归）；稳定后再考虑微调。

交付物：`.histwrite/learn/episodes/*.jsonl`、`.histwrite/learn/datasets/*.jsonl`、`prompt-pack.json`（如项目采用）。

## 11) 写完“大修”仍不满意：升级流程（把问题打穿）

1. 先定位“不满意的类型”（只选 1–2 类作为本轮主攻）：
   - A 论题/范围/取舍跑偏（回到蓝图与材料权重）
   - B 段落推进不顺、论证不一气呵成（回到大纲与段落级判断句）
   - C 史论结合失衡（材料堆砌或论述空转）
   - D 去 AI 硬门槛（1/2/3/4/6）仍回潮
   - E 学术规范门面（引注体例/本地路径/档案字段）
2. 做一份 `导出/revise_brief.md`（下一轮的“唯一施工单”）：
   - 写明本轮只改什么、不改什么
   - 每个问题给 2–4 个“原句/原段落片段”（短摘录即可）+ 你希望的改法
3. 先用 500–900 字小样本 A/B 验证方向（能对就扩到全文；方向错就立刻回滚，不浪费全篇重写额度）。
4. 若连续两轮仍不满意：停止打补丁，直接做“结构性重写”（重写大纲→重分配证据卡→按新推进重写关键部分），并把这两轮新增偏好沉淀回长期记忆与 style-profile。
