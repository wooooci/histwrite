## 2. 不可改动的“事实与约束”

> 这部分用于防止精修时误改含义或引入新事实。

- 不得新增事实/数据/引文/页码/档案号：是（强制）
- 关键术语表（必须统一写法）：<<术语A=…；术语B=…；人名/地名译名…>>
- 必须保留的句子/段落（原文粘贴）：<<如有就贴；没有写“无”>>
- 绝对不能出现的内容：<<例如：免责声明/模型自述/“作为AI”…>>

## 3. 这轮的“最重要目标”（优先级最高）

把你刚刚那段要求压缩成可执行版本（已预填，你可补充/删改）：

1) **连接词与句式模板彻底打散**：同一种逻辑关系（转折/递进/因果/并列）不能总靠同一套句式；同一连接词就算重复出现，形态/断句/句长/语序也必须变化。
2) **线性叙述体验**：用信息推进与视角移动把段落串起来，避免“块状堆叠”的叙事块。
3) **强制降频的高重复结构**（出现越多扣分越重）：
   - 当…时，就…
   - 并不是…而是… / 不是…而是… / 不在…而是…
   - 越…就越…
   - ……，反而…… / ……，却……
   - 既…也… / ……，并…… / ……，也……
   - 并列名词串：X、Y与Z / X与Y / X、Y、Z
   - 被动模板：X 被 Y …（或同类被动句堆叠）
   - 单字连接词密集（尤其“而/却/亦”等连续承接）
   - “概括 + 冒号 + 多句解释块”反复出现（需要拆散并融入段落推进）

## 4. 允许的改写手段（请勾选）

> 你允许我用哪些“结构性动作”来打散句式：不勾选默认允许。

- [ ] 允许拆句/合句（为了变换节奏与语序）
- [ ] 允许更换主语/视角（同一事实换叙述入口）
- [ ] 允许调整句内信息顺序（先因后果 / 先结论后限定）
- [ ] 允许把显式连接词改成隐式衔接（用语义推进代替“然而/因此/同时”）
- [ ] 允许把并列串改成“逐一展开”或“分句递进”

- 输出格式：<<Markdown｜纯文本>>

## 7. 评分与验收（去 AI 感专用）

> 你可以直接把本仓库的 `extensions/histwrite/templates/eval-rubric.zh.md` 作为 judge 的 `prompts.evalRubric`。

本轮额外加严（默认启用，可改）：

- 同一连接词（如“却/反而/然而/不过/因此/同时/此外/于是”）**在相邻段落或近距离内**若以同构句式重复出现，判为硬扣分（即使含义不同也要变形）。
- 对“概括+冒号+解释块”“并列名词串”“被动模板”“单字连接词密集”四类结构，采用更高权重惩罚。

## 8. 一键执行建议（我会按这个跑）

> 全文较长建议按章拆分，多次运行，减少一次性上下文压力。

1) 把全文（或某章）放到文件：`paper-latest.md`
2) （可选）把 rubric 正文写入 `prompt-pack.json` 的 `prompts.evalRubric`
3) 运行：`/histwrite on`（如未开启）
4) 精修选优：`/histwrite polish best 4 file paper-latest.md`
5) 如仍有“句式同构重复”，再跑一轮（或我会根据 episode 记录调整 prompts）
