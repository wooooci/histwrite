# 隐私与公开边界

本仓库公开发布时必须满足以下条件：

## 不得出现的内容

- 个人邮箱、真实姓名、私人用户名
- 本机绝对路径示例
- 学校/机构私有代理入口与定制数据库入口
- cookie、token、session、API key、密码、浏览器凭据
- 私人工作日志、会话记录、缓存目录、下载收件箱

## Git 记录策略

- 新仓库使用**公开安全身份**提交
- 不直接导入上游私有仓库的原始 Git 历史
- 公开保留方式为：
  - 新仓库中的结构化提交历史
  - `docs/history/upstream-timeline.md` 中的脱敏演化记录

## 发布前必须执行

```bash
pnpm privacy:scan
```

若扫描结果包含私人路径、邮箱或机构入口，必须先修复再提交。
